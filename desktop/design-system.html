Tokens

Sora primary, warm sand secondary.

Sora / 空色 is the calm focus color for selected worktrees and primary actions. Warm Sand / 砂色 is the supporting structure color for Git connectors, quiet navigation, and empty surfaces. Status colors stay separate so dirty/conflict states remain truthful.

State model

Truth before optimism.

Try the danger palette against approval state. Kurenai is calmer and more serious; Persimmon is warmer and more action-oriented.

StateVisualTruth rule
CleanCleanGit status has no local changes.
DirtyDirtyShow dirty state until Git confirms otherwise.
ConflictConflictNever hide merge/rebase conflicts behind agent progress.
Write lockWrite lock: AgentOnly one write-mode owner per worktree.
ApprovalApproval requiredPermission escalation is explicit and durable until resolved.
Interactive state note
Clean means Git has confirmed there are no local file changes. Do not infer clean from agent completion alone.

Components

Small primitives before framework commitment.

Component boundaries follow product truth: a worktree is the core object, agent UI hangs from a worktree, and approvals/artifacts/runs are inspectable state rather than decoration.

Atoms01

Lowest-level visual primitives. They should not know about Git, agents, or approvals.

Molecules02

Small combinations that express one compact state or control.

Organisms03

Product-aware regions. Start implementation here only after the smaller contracts are stable.

Templates04

Layout contracts for arranging product regions without binding to real data yet.

Pages05

Routable surfaces that connect layout, fake data, and later AgentBridge service state.

Breakpoints

Viewport changes behavior, not truth.

Phone

Tree becomes list; inspector moves below selected card; no drag-only actions.

Tablet

Sidebar collapses; canvas remains scrollable; inspector can become drawer.

Desktop

Full sidebar, tree canvas, right inspector, bottom timeline.

Wide desktop

More canvas space and denser comparison, not viewport-scaled typography.

Decisions

Rules that keep the dashboard honest.

DoUse Sora primary for the selected worktree and active write/focus path.
Don'tUse primary color to decorate every panel; it weakens selection meaning.
DoUse Warm Sand secondary for Git structure and quiet operations.
Don'tUse secondary color for dirty, conflict, or approval states.
DoShow empty states truthfully, like Spectra's no-active-changes panel.
Don'tImply an agent is running, writing, or approved without source-backed state.
